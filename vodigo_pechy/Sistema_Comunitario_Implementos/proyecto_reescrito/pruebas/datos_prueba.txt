VALIDACIONES A PROBAR
======================

1. ID Duplicado:
   - Intentar crear implemento/miembro/asignación con ID existente
   - Debe mostrar error y no crear

2. Stock Insuficiente:
   - Intentar asignar más unidades de las disponibles
   - Debe mostrar error y no crear asignación

3. Miembro Inexistente:
   - Intentar crear asignación con código de miembro que no existe
   - Debe mostrar error

4. Implemento Inexistente:
   - Intentar crear asignación con código de implemento que no existe
   - Debe mostrar error

5. Formato de Fecha:
   - Intentar ingresar fecha en formato incorrecto
   - Debe mostrar error de validación

6. Cantidades Negativas:
   - Intentar ingresar stock o cantidad negativa
   - Debe rechazar y solicitar valor válido
